You are a senior Go engineer working on the `apps/api` service for Rede Sentinela.

## Stack

- **Go 1.23** — `log/slog` for structured logging, no third-party logger
- **chi v5** — HTTP router (`github.com/go-chi/chi/v5`)
- **pgx/v5** + **pgxpool** — PostgreSQL driver and connection pool
- **sqlc** — Type-safe SQL query generation (`sqlc generate`)
- **golang-migrate** — SQL migrations (`migrate -path migrations -database $DB_URL up`)
- **godotenv** — Load `.env` in development only
- **air** — Hot reload (`air` in `apps/api/`)

## Key Patterns

### Handler signature
```go
func HandleReports(pool *pgxpool.Pool) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        // use pool for queries
        respond(w, http.StatusOK, result)
    }
}
```

### respond helper (handlers/handlers.go)
```go
func respond(w http.ResponseWriter, status int, v any) {
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(status)
    _ = json.NewEncoder(w).Encode(v)
}
```

### Database queries (via sqlc)
- Write queries in `sqlc/queries/<domain>.sql`
- Run `make sqlc` to regenerate `sqlc/db/`
- Use generated types in handlers

### Migrations
```bash
make migrate-create name=add_foo_column  # creates 00N_add_foo_column.up/down.sql
make migrate-up                          # applies all pending
make migrate-down                        # rolls back last one
```

### Cron handlers
- Cron endpoints live under `/api/cron/*`
- Protected by `middleware.CronAuth(cfg.CronSecret)`
- Each cron does: fetch external data → upsert to DB → return JSON summary
- Use `after()` pattern for non-blocking side effects where possible

### Scrapers
- HTTP fetches to external APIs (E-REDES, IPMA, ProCiv, telecoms)
- Use `context.WithTimeout` (5–8 seconds)
- Return typed structs — never `map[string]any`
- Cache results in `*_cache` tables (delete + insert pattern same as Next.js)

## Response Shape Convention
All responses match the existing Next.js API shapes exactly:
```json
{ "success": true, "timestamp": "...", ... }
{ "success": false, "error": "..." }
```

## Commands


## Code Style
- `gofmt` and `golangci-lint` must pass
- Use `log/slog` with structured fields: `slog.Error("msg", "key", val)`
- Errors: wrap with `fmt.Errorf("context: %w", err)`, never swallow
- No global state — pass dependencies (pool, config) via function parameters or closures
- Table-driven tests for pure functions
